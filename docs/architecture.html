SquadPad Architecture

Understanding connection modes — Original BombSquad Remote vs SquadPad

Original BombSquad Remote (iOS App)

The official iOS remote connects directly to BombSquad over UDP. No middleman — the phone talks to the game natively.

Key: Zero setup. Phone discovers BombSquad automatically via UDP broadcast, connects directly. No host app, no relay, no port forwarding. Works only on the same LAN.

Connection Lifecycle Comparison

Step-by-step flow of what happens from the player's perspective in each mode.

sequenceDiagram
  participant P as 📱 Player
  participant R as ☁️ Relay
  participant H as 🖥️ Host App
  participant B as 🎮 BombSquad

  rect rgba(187, 154, 247, 0.08)
    Note over P,B: Original BombSquad Remote (iOS) — Direct UDP
    P->>B: UDP Broadcast GAME_QUERY (:43210)
    B-->>P: GAME_RESPONSE (game name)
    P->>B: ID_REQUEST (name, protocol v121)
    B-->>P: ID_RESPONSE (player ID)
    loop Every input change
      P->>B: STATE2 (3 bytes: buttons, h, v)
      B-->>P: STATE_ACK
    end
  end

  rect rgba(122, 162, 247, 0.08)
    Note over P,B: SquadPad Relay Mode — WebSocket → Relay → Host → UDP
    H->>R: WS: {type: "host"}
    R-->>H: {type: "room", code: "swift cobra"}
    P->>R: WS: {type: "join", room: "swift cobra", name: "Alex"}
    R-->>P: {type: "joined", playerId: 0}
    R->>H: {type: "player_joined", playerId: 0, name: "Alex"}
    H->>B: ID_REQUEST ("Alex#sp0")
    B-->>H: ID_RESPONSE
    loop Every input change
      P->>R: Binary [buttons, h, v]
      R->>H: Binary [playerIdx, buttons, h, v]
      H->>B: STATE2 packet
      B-->>H: STATE_ACK
    end
  end

  rect rgba(158, 206, 106, 0.08)
    Note over P,B: SquadPad LAN Mode — WebSocket → Host → UDP
    P->>H: WS: {name: "Alex"} (ws://HOST:43211)
    H-->>P: {type: "connected", playerId: 0}
    H->>B: ID_REQUEST ("Alex#sp0")
    B-->>H: ID_RESPONSE
    loop Every input change
      P->>H: Binary [buttons, h, v]
      H->>B: STATE2 packet
      B-->>H: STATE_ACK
    end
  end

The Core Constraint

Original Native App

  • Direct UDP to BombSquad
  • Auto-discovers games on LAN
  • No middleman needed
  • iOS only, requires app install
  • LAN only — no internet play

SquadPad Browser

  • Cannot use UDP (browser limitation)
  • Must use WebSocket as transport
  • Needs a bridge app (Tauri) to convert WS→UDP
  • Works on any device with a browser
  • Relay enables internet play

🤔 The Gap

  • LAN discovery doesn't work from browser
  • Player must know host IP or room code
  • Original app: zero-config on LAN
  • SquadPad LAN: requires manual IP
  • Could the host broadcast a join URL?
The fundamental question: The original app connects directly to BombSquad because it speaks native UDP. A browser can't do that. So SquadPad always needs a host app as a bridge. The question is: how do we make the LAN experience as seamless as the original? Auto-discovery? QR codes? mDNS? Or is the relay-first approach (room codes) good enough for all scenarios?
